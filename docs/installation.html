Deployment Guide

Get OpenTicket running locally or in your production environment in under 5 minutes.

Alternative: Docker Deployment

The fastest way to deploy OpenTicket to production is using our pre-configured Docker Compose stack.

git clone https://github.com/Cyber-Sec-Space/openticket.git
cd openticket
docker-compose up -d --build

This builds the production image, spins up a PostgreSQL container, auto-migrates the database schema, and exposes the app on port 3000.

1. Clone the Repository (Manual Deployment)

Start by cloning the active codebase from GitHub using your preferred Git client.

git clone https://github.com/Cyber-Sec-Space/openticket.git
cd openticket

2. Configure the Environment

We use Prisma to manage the PostgreSQL database. Ensure you have a running Postgres instance, then duplicate the environment template.

cp .env.example .env

Edit `.env` and fill in your `DATABASE_URL` and `NEXTAUTH_SECRET`.

3. Initialize Database

Install dependencies and push the Prisma schema to your target database. This will build all required tables (Incidents, Assets, Users, PluginState).

npm install
npx prisma generate
npx prisma db push

4. Start the Application

Run the development server. OpenTicket will be accessible globally via your configured localhost port.

npm run dev
