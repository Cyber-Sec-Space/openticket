Deployment Guide

Get OpenTicket running locally or in your production environment in under 5 minutes.

Alternative: Production Docker Deployment (HA)

The fastest way to deploy OpenTicket to production is using our pre-configured Docker Compose stack. It natively embeds a PgBouncer Connection Pool for massive scalability and encapsulates schema upgrades asynchronously inside a secure transient Migrator Container.

git clone https://github.com/Cyber-Sec-Space/openticket.git
cd openticket
docker-compose up -d --build

This builds the production image, spins up a PostgreSQL container, auto-migrates the database schema, and exposes the app on port 3000.

1. Clone the Repository (Manual Deployment)

Start by cloning the active codebase from GitHub using your preferred Git client.

git clone https://github.com/Cyber-Sec-Space/openticket.git
cd openticket

2. Configure the Environment

We use Prisma to manage the PostgreSQL database. Ensure you have a running Postgres instance, then duplicate the environment template.

cp .env.example .env

Edit `.env` and fill in your `DATABASE_URL` and `NEXTAUTH_SECRET`.

3. Initialize Database

Install dependencies and prepare the database. Choose between a Clean Installation (Empty Database) or a Demo Installation (pre-loaded with 150 incidents, assets, and users).

Option A: Clean Version (Production Ready)

npm install
npx prisma generate
npx prisma db push

Option B: With Demo Info (Sandbox Data)

npm install
npx prisma generate
npx prisma db push
npx prisma db seed

4. Start the Application

Run the development server. OpenTicket will be accessible globally via your configured localhost port.

npm run dev

⬆️ Cross-Version Safe Upgrades (0.3.0 -> 0.5.0)

If you are upgrading from legacy states to the new v0.5.0 Plugin & PgBouncer architecture, you must run our unified idempotent migration utility. It safely chains legacy schema mappings before natively applying `0.5.0` plugin privileges without dropping user roles.

git pull origin main
npm install
npm run migrate:prod
