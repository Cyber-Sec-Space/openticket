Integrate seamlessly with your existing SOC infrastructure using zero-config extensions.

The Plugin Architecture

OpenTicket plugins are natively built into the Next.js runtime. Since this is an edge-compatible application, plugins are compiled at build-time to ensure maximum performance and security.

Plugin Repository: All plugins are located in the source code under src/plugins/.

How to Develop a Plugin

A plugin is simply a TypeScript object that implements the OpenTicketPlugin interface. You can tap into the Hook Engine's events to trigger external systems (e.g., Slack Webhooks, Jira, or custom APIs). 

import { OpenTicketPlugin } from '@/types/plugin';

export const myCustomPlugin: OpenTicketPlugin = {
  id: 'my-custom-plugin',
  name: 'Custom Integration',
  description: 'Sends alerts to my custom API.',
  hooks: {
    onIncidentCreated: async (incident, config) => {
      await fetch('https://api.mycorp.com/alerts', {
        method: 'POST',
        body: JSON.stringify(incident)
      });
    }
  }
};

How to Register a Plugin

Once your plugin is written, you must statically register it into the system's active runtime. Open src/plugins/index.ts and add your plugin to the activePlugins array.

import { myCustomPlugin } from './my-custom/index';

export const activePlugins: OpenTicketPlugin[] = [
  // Add your plugin here
  myCustomPlugin
];

Rebuild the application via `npm run build` or `docker-compose build` to apply your changes.
