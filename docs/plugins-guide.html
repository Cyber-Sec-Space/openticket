The Hybrid Plugin Architecture

OpenTicket plugins operate as dynamic external modules. Since this is an edge-compatible application, plugins interact purely with the centralized Hook Engine and EventBus rather than polluting core logic.

Hybrid Orchestration: You no longer need to write code to install plugins! Administrators can browse the Plugin Store inside their dashboard and install remote capabilities directly via our Server-Side Registry Orchestration logic.

How to Develop a Zero-Trust Plugin (SDK v0.5.0+)

A plugin is simply a TypeScript object conforming to the OpenTicketPlugin interface. With our new Zero-Trust architecture, plugins must explicitly declare the privileges they require via the requestedPermissions array. Administrators will review and consent to these permissions via an OAuth-style UI before the plugin can interact with the SOC Database.

import { OpenTicketPlugin } from '@openticket/core';

export const myJiraIntegration: OpenTicketPlugin = {
  manifest: {
    id: 'jira-bridge',
    name: 'Jira Two-Way Sync',
    description: 'Automatically syncs high-severity incidents to Jira.',
    version: '1.0.0',
    requestedPermissions: ['VIEW_INCIDENTS_ALL', 'ADD_COMMENTS'] // Must be explicitly declared
  },
  hooks: {
    onIncidentCreated: async (payload, config, context) => {
      if (payload.severity !== 'CRITICAL') return;
      
      // 1. Call external API securely using encrypted config secrets
      const res = await fetch('https://api.mycorp.atlassian.net/rest/api/2/issue', {
        method: 'POST',
        headers: { 'Authorization': `Bearer ${config.jiraApiToken}` },
        body: JSON.stringify({ summary: payload.title })
      });
      
      const jiraData = await res.json();

      // 2. Use the isolated Sandbox context to safely write back to the SOC DB
      await context.api.addComment(payload.id, `Jira Ticket Created: ${jiraData.key}`);
    }
  }
};

Understanding the execution Context & Cryptography

In standard monolithic platforms, plugins are dangerous because they can import raw database modules. In OpenTicket 0.5.0+, you are legally restricted interacting only via the context.api Sandbox constraint. If your plugin attempts to call context.api.deleteIncident() but did not request DELETE_INCIDENTS in its manifest, the Engine will unconditionally reject the Promise.

Furthermore, all credentials (such as config.jiraApiToken mentioned above) are encrypted at rest using AES-256-GCM. Your server secret entropy ensures that if the database is backed-up or compromised, third-party authentication tokens remain secure.

How to Register Custom Plugins Manually

If you prefer not to use the OpenTicket Registry and want to deploy a highly-sensitive internal air-gapped integration, you can still statically register it. Open src/plugins/index.ts and add your plugin to the activePlugins array. 

import { myCustomPlugin } from './my-custom/index';

export const activePlugins: OpenTicketPlugin[] = [
  // Add your plugin here
  myCustomPlugin
];

Rebuild the application via `npm run build` or `docker-compose build` to apply your changes manually.
